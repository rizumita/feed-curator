# Curate Articles

Read uncurated articles, score their relevance, write summaries, assign tags, and generate a Markdown digest. Uses reading history to personalize scoring.

## Steps

1. Run `bun src/cli.ts config language` to check the user's preferred language. If set, write all summaries and the digest in that language. If not set, use English.
2. Run `bun src/cli.ts profile --prompt` to get the user's reading preferences. Use this to adjust scores:
   - **Boost** scores for articles matching preferred tags and sources
   - **Lower** scores for articles matching ignored tags
   - If no profile data exists yet (no read history), skip this step and use default scoring
3. Run `bun src/cli.ts articles --uncurated --json` to get uncurated articles as JSON
4. For each article, evaluate its title and content to determine:
   - **Score** (0.0-1.0): Based on novelty, technical depth, practical utility, and breadth of interest. **Adjust based on user profile from step 2.**
   - **Summary** (2-3 sentences): Capture the key insight of the article. **Write the summary in the user's configured language.**
   - **Tags** (comma-separated): Assign 1-3 short topic tags in English (e.g. `agents,mcp,security`). Use consistent, reusable tag names across articles.
5. Run `bun src/cli.ts update <id> --score <score> --summary "<summary>" --tags "<tags>"` for each article
6. After all articles are updated, generate a Markdown digest file at `output/digest-YYYY-MM-DD.md` sorted by score descending. **Write the digest headings and summaries in the user's configured language.** Use this format:

```markdown
# Feed Digest - YYYY-MM-DD

## Top Articles

### 1. Article Title (Score: 0.95)
**Source:** https://example.com/article
**Tags:** agents, mcp
**Summary:** Summary in the user's preferred language...

---

*Generated by feed-curator. N articles curated.*
```

7. Create the `output/` directory if it doesn't exist before writing the file
8. Report the results to the user
